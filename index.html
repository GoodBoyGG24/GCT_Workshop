ເຄີິຍຂຽນໂປຣແກຣມຮ່ວມກັບຄົນອື່ນ ຫຼື ບໍ່? ປົກກະຕິແລ້ວ, ເຮົາຈະເຮັດແນວໃດຖ້າຫາກຢາກແບ່ງກັນເຮັດວຽກ ແຍກກັນເຮັດຄົນລະ Function ຫຼື Feature? ເວລາແກ້ໄຂ Code ໃນສ່ວນດຽວກັນ ຫຼື ຕໍ່ກັນຈະແກ້ໄຂແນວໃດ?

ຫາກວ່າຢາກທົດສອບຫຍັງບາງຢ່າງແລ້ວຢ້ານ code ພັງ, ຖ້າ Code ໄວ້ຫຼາຍໆອັນ ແລະ ເມື່ອຈະເອົານຳມາໃຊ້ກໍບໍ່ຮູ້ວ່າໂຕໃດເປັນ Version ລ່າສຸດ.

ບັນຫາເຫຼົ້ານີ້ຈະໝົດໄປຖ້າຫາກຫັນມານຳໃຊ້ Git Version Control + GitHub
Git ແມ່ນ......?

Git ເປັນ version control System ທີ່ເປັນ Open source ໂດຍມັນຈະໃຊ້ຕິດຕາມປະຫວັດ ຫຼື ກວດສອບການແກ້ໄຂ File ຫຼື Source code ເຊິ່ງຈະກວດສອບໄດ້ວ່າຈຸດໃດຖືກແກ້ໄຂ ແລະ ພ້ອມບອກວ່າຖືກແກ້ໄຂເວລາໃດ ແລະ ໃຜເປັນຜູ້ແກ້ໄຂ.

ຕິດຕັ້ງ Git Version Control

ສຳຫຼັບການຕິດຕັ້ງກໍສາມາດເຂົ້າໄປ download ໄດ້ໃນ Website ຂອງ Git ເລີຍ Git-Download


ວິທີຕິດຕັ້ງ : ກົດເຂົ້າ URL ທາງເທິງ ແລະ ເລືອກລະບົບປະຕິດບັດການ ແລ້ວກົດດາວໂຫຼດ

ຈາກນັ້ນກໍເປີດ File ເພື່ອຕຶດຕັ້ງ....

ເມື່ອຕຶິດຕັັ້ງສຳເລັດແລ້ວ ລອງໃຊ້ຄຳສັ່ງ git --version ໃນ terminal ຫຼື cmd ເພື່ອເປັນການກວດສອບວ່າຕິດຕັ້ງ git ຜ່ານ ຫຼື ບໍ່

Config git user....

ຫຼັງການຈາກຕິດຕັ້ງແລ້ວຈະຕ້ອງມາ Set ຄ່າເລັກໜ້ອຍເພື່ອກຽມພ້ອມໃຊ້ງານ

  • ເປີດ Terminal ຫຼື cmd
  • git config --global user.email "ອີເມວຂອງຜູ້ໃຊ້"
  • git config --global user.name "ຊື່ຂອງຜູ້ໃຊ້"
    •  #ຢ່າລືມໃສ່ "..." ໃຫ້ຊື່ ແລະ ອີເມວຂອງຜູ້ໃຊ້

    GitHub ແມ່ນ......?

    ເປັນ Website ທີ່ໃຫ້ບໍລິການ Git (Version Control Repository) ເຊິ່ງຈະໃຫ້ບໍລິການ Cloud ໃຫ້ເຮົາສາມາດນຳ Project ຫຼື Source code ໄປຝາກໄວ້ໃນ Server ໄດ້ແບບ FREE ແລະ ຍັງສາມາດເປີດເປັນ Private ຫຼື Public ເພື່ອໃຫ້ຄົນອື່ນມາເບິ່ງ Source code ຂອງເຮົາໄດ້.

    ສະໝັກບັນຊີ GitHub

    ຖ້າວ່າທ່ານມີບັນຊີຂອງ GitHub ຢູ່ແລ້ວ, ທ່ານສາມາດລົງທະບຽບເຂົ້າ Website ຂອງ GitHub ໄດ້ເລີຍ.
    ແຕ່ຖ້າທ່ານບໍ່ທັນມີບັນຊີຂອງ GitHub ທ່ານຈຳເປັນຕ້ອງສະໝັກສະມາຊິກກ່ອນໄດ້ທີ່ GitHub-register

    ຄຳສັ່ງພື້ນຖານຂອງມີຫຍັງແດ່?

    ມາເບິ່ງຄຳສັ່ງພື້ນຖານຂອງ git ທີ່ເຫັນມັກຖືກນຳໃຊ້

    • git init

      • ສ້າງ Local Repository ຫຼື file .git


      ການເຮັດວຽກຄືໃຫ້ git ມັນເຮັດວຽກຢູ່ແຕ້ໃນສ່ວນຂອງ folder ທີ່ຊຶ່ວ່າ GitCT-Web ນັ້ນເອງ

    • git add

      • ເພີ່ມ file ເຂົ້າສູ່ stage ຫຼື ສະຖານະການປ່ຽນແປງ ເຊິ່ງວິທີການກຳນົດ file ເຂົ້າ stage ຈະມີຫຼາຍແບບດັ່ງນີ້:

      1. ເພີ່ມທັງໝົດທຸກ file ຫຼື folder

        2. git add .


      2. ເພື່ອສະເພາະບາງ file

        3. git add ຊື່file.ນາມສະກຸນ


      3. ເພື່ອສະເພາະ file ທີ່ເປັນນາມສະກຸນທິ່ກຳນົດທັງໝົດ

        4. git add *.ນາມສະກຸນ


    • git commit -m "comment"

      • ຢັ້ງຢືນການປ່ຽນແປງ file ທີ່ຖືກ add ເຂົ້າ stage


      -m "ຂໍ້ຄວາມ" ບ່ອນຂໍ້ຄວາມຈຳເປັນຕ້ອງໄດ້ໃສ່ ເພາະຈະເປັນການແຈ້ງຄົນໃນທີມວ່າເຮົາໄດ້ເພີ່ມ ຫຼື ແກ້ໄຂ code ສ່ວນໃດ

    • git reset

      • ເປັນການຍ້ອນ version file ກ່ອນທີ່ຈະຖືກ add ເຂົ້າ stage


    • git push

      • ເປັນການສົ່ງ file ທີ່ຖືກ commit ເຂົ້າໄປທີ່ remote repository


      orgin ເປັນຊື່ຂອງ remote ແລະ main ເປັນຊື່ git branch ທີ່ຖຶກສ້າງຂື້ນ

    • git clone

      • ເປັນການດາວໂຫຼດ Project ມາຈາກ Project ຫຼັກ, ເຊິ່ງເຮົາສາມາດແກ້ໄຂ ຫຼື ປ່ຽນແປງ File ໃນ Project ນັ້ນໄດ້ໂດຍຈະບໍ່ມີຜົນກະທົບຕໍ່ Project ຫຼັກ


      Project ທີ່ເຮົາ clone ມາເອີ້ນວ່າ local repository
      Project ຫຼັກ ເອີ້ນວ່າ remote repository.
